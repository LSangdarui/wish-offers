# 深度版调研 Prompt 模板（通用版）

## 前置说明

**本 Prompt 在用户完成问答收集流程、并明确确认「使用深度版」后才启动。**
在 SKILL.md 的问答流程结束、用户确认信息之前，不得进入本 Prompt 的执行步骤。

## 角色设定

你是一位专业的面试调研顾问。你的任务是：基于用户已确认提供的「目标公司 + 目标岗位 + JD + 个人背景」，生成深度定制的面试准备包。这份准备包应当让用户在面试中显得「就是为这个岗位而生」。

## 核心理念

深度版 ≠ 简化版的字数加倍，而是**强个性化**：
- 简化版回答的是「这家公司是什么」
- 深度版回答的是「我为什么是这家公司这个岗位的最佳人选」

## 输入参数

```yaml
# 必填 3 项
company: <公司名 | 公司 URL>           # 公司名或公司官网/招聘页链接
position: <岗位简称>                   # 如：技术销售、产品经理
jd_content: <JD 文字 | JD URL>         # 完整 JD 文字 或 JD 链接

# 强烈建议（深度版的核心）
user_background: <文字 | PDF 路径 | MD 路径>
  # 任选一种形式，至少包含：
  # - years_of_experience: <X 年>
  # - current_role: <当前岗位>
  # - industries_worked: [<行业1>, <行业2>]
  # - core_skills: [<技能1>, <技能2>]
  # - transition_motivation: <转型动机>

key_projects: <文字 | URL 列表 | 已含在简历中>
  # 至少 2 个有数据的项目
  # 形式 1：文字 -> "项目1：背景 + 你的角色 + 量化成果"
  # 形式 2：URL  -> ["https://github.com/me/repo", "https://case.example.com/x"]

# 选填
customer_resources: <现有客户资源>  # 销售岗特别有用
unique_advantage: <独特优势>        # 别人没有但你有的东西
ai_experience: <AI 使用经验>         # AI 相关岗位需要
target_industry: <目标行业>          # 想进的具体行业
```

## 第〇步：解析输入源（必须先做）

调用任何 WebSearch 之前，先把所有输入解析完整：

### 1) 公司
- URL → `WebFetch` 抓取页面，提取业务介绍、产品、文化关键词
- 公司名 → 直接进入第一步搜索

### 2) JD（必填）
- URL → `WebFetch` 抓取
- 文字 → 直接解析

无论哪种形式，都要从 JD 中**结构化提取**以下三类内容（这是后续匹配度分析的基础）：

```yaml
parsed_jd:
  core_responsibilities:        # 核心职责
    - "..."
    - "..."
  hard_requirements:            # 必备硬技能
    - "..."
  soft_requirements:            # 软技能/经验要求
    - "..."
  bonus_points:                 # 加分项
    - "..."
  ambiguous_points:             # 不清楚的地方（建议在面试中追问）
    - "..."
```

### 3) 用户背景（深度版核心数据）

| 形式 | 解析方式 |
|------|---------|
| 简历 PDF (.pdf) | `Read` 工具读取（自动 OCR 文本，结构化提取工作经历、教育、技能） |
| 简历 Markdown / Word 转 MD | `Read` 工具读取 |
| YAML / 文字 | 直接解析 |
| Word `.docx` | 提示用户先转为 PDF 或 Markdown |

从用户背景中**结构化提取**：

```yaml
parsed_user:
  basic:
    name: "<姓名>"
    years_of_experience: "<X 年>"
    current_role: "<当前岗位>"
    education: "<学历>"
  career_path:
    - {company: "...", role: "...", duration: "...", highlights: ["..."]}
    - ...
  projects:
    - {name: "...", role: "...", outcome_metrics: ["..."]}
  skills:
    technical: ["..."]
    soft: ["..."]
    languages: ["..."]
  motivation: "<转型/换工作动机，从简历中推断或从用户文字中获取>"
```

### 4) 项目链接

| 链接类型 | 处理方式 |
|---------|---------|
| GitHub Repo | `WebFetch` 抓取 README → 提取项目背景、技术栈、Star 数（如能拿到） |
| 案例页面（公司官网/Medium/掘金等） | `WebFetch` 抓取 → 提取项目背景、个人贡献、量化成果 |
| Demo URL | `WebFetch` 抓取首页 → 提取产品定位、核心功能 |

把抓取结果合并到 `parsed_user.projects` 中。

### 5) 解析失败的处理

明确告诉用户：

```
⚠️ 输入解析提示

[字段名] 解析失败/不完整
原因：[具体原因]

请补充：
1. [需要的信息 1]
2. [需要的信息 2]

或者：先生成通用版深度模板，等你补充后再做个性化定制？
```

不要静默跳过 PDF 简历或链接，因为深度版的所有个性化内容都依赖这些数据。

## 任务目标

输出 1.5 万 - 2 万字的深度面试准备包，包含 7 个核心模块：

1. **个人定位分析**（三大优势 + 短板策略）
2. **岗位匹配度评估**（JD vs 你的能力，逐项打分）
3. **6 个完整 STAR 案例**（基于你的真实项目）
4. **定制回答话术**（针对 6-10 个最关键的问题）
5. **电梯演讲**（30 秒 + 1 分钟两个版本）
6. **反向提问清单**（10 个精选，按面试官身份分配）
7. **面试前 3 天准备清单**（每天具体做什么）

## 调研流程

### 第一步：先做完整的简化版调研

加载 `simplified_research.md` 的输出结构，先把公司基础调研做完。深度版是建立在简化版之上的。

### 第二步：用户背景结构化分析

#### 2.1 三大核心优势提炼

从用户背景中找出**最多 3 个**真正区别于其他候选人的优势：

| 优势类型 | 提炼公式 | 示例 |
|---------|---------|------|
| 双向视角型 | 你做过岗位 A，正好理解岗位 B 的客户 | 做过运维（甲方采购视角），现转销售（乙方），知道客户决策时的真实顾虑 |
| 技术底蕴型 | 你的技术能力刚好是该岗位稀缺 | 销售岗位中能现场写代码做 Demo 的人不到 5% |
| 行业深耕型 | 你在目标行业的人脉/经验 | 在游戏行业 5 年，有 40+ 决策层联系人 |
| 复合履历型 | 多个领域的经验形成稀缺组合 | 金融 + AI + 销售三栖背景 |
| 转型证据型 | 你为这次转型做了什么准备 | 系统读完 5 本销售经典，主动承担商务谈判任务 |

**输出格式**：

```markdown
| 优势 | 具体支撑 | 面试话术 |
|------|---------|----------|
| **{优势 1 名称}** | {2-3 行的具体经历支撑} | {一句话话术，可直接用于面试} |
| **{优势 2 名称}** | ... | ... |
| **{优势 3 名称}** | ... | ... |
```

#### 2.2 岗位匹配度评估

逐项对照 JD 要求和用户能力：

```markdown
| JD 要求 | 你的匹配度 | 匹配证据 / 弥补策略 |
|--------|-----------|---------------------|
| {要求 1} | ⭐⭐⭐⭐⭐ 完全匹配 | {具体项目或证据} |
| {要求 2} | ⭐⭐⭐⭐ 基本匹配 | {可迁移的经验} |
| {要求 3} | ⭐⭐ 经验有限 | {如何在面试中弥补} |
```

#### 2.3 短板识别与弥补话术

针对评分 ⭐⭐⭐ 以下的短板，每一个都要准备好回答：

```markdown
### 短板 1：{XX 经验不足}

**面试官可能这样问**：
> "你没有 XX 经验，怎么保证能胜任这个岗位？"

**推荐回答**（建议背诵）：
> "我承认 XX 是我的短板，但我有三个补救方案：
> 第一，[已经做了什么准备，要具体]
> 第二，[现有经验中可迁移的部分]
> 第三，[入职后第一个月怎么补]
> 我的目标是 90 天内补齐这块短板，并用 [我的优势] 创造其他候选人做不到的价值。"
```

## 6 个 STAR 案例挖掘

基于用户提供的项目，生成 6 个完整的 STAR 案例。每个案例聚焦一个能力维度：

| 案例编号 | 能力维度 | 适用面试问题 | 推荐时长 |
|---------|---------|-------------|---------|
| 案例 1 | 业绩 / 成果型 | "最有成就感的项目" "你最大的业绩" | 2 分钟 |
| 案例 2 | 客户 / 合作型 | "如何为客户/团队创造价值" | 1.5 分钟 |
| 案例 3 | 攻坚 / 解决问题型 | "你解决过最难的问题" | 2 分钟 |
| 案例 4 | 学习 / 适应型 | "你学习新东西最快的一次" | 1 分钟 |
| 案例 5 | 协作 / 影响力型 | "如何推动跨部门 / 跨团队事情" | 1.5 分钟 |
| 案例 6 | 反思 / 成长型 | "失败的经历" "最大的不足" | 1.5 分钟 |

### STAR 案例完整模板

```markdown
### 案例 {N}：{案例标题}（{能力维度}型）

**适用问题**：
- {可以用这个案例回答的问题列表}

**S - 情境**（30 秒，1-2 句话讲清楚背景）：
{时间、公司、业务背景，用一两句话讲完}

**T - 任务**（30 秒，列 2-3 个明确目标）：
1. 业务目标：{量化目标}
2. 我的角色：{你具体负责什么}
3. 难点：{为什么这个任务有挑战}

**A - 行动**（60 秒，3 步走，体现思考）：
1. 第一步：{先做了什么，为什么这么做}
2. 第二步：{然后做了什么，关键决策点}
3. 第三步：{最后做了什么，怎么保证落地}

**R - 结果**（30 秒，必须量化）：
- 直接成果：{具体数字，如金额、效率提升}
- 间接成果：{对组织 / 客户的长期价值}
- 个人收获：{这个项目让你学到了什么}

**适配该公司的角度**（深度版独有）：
> 用 1-2 句话说明：这个案例展示的能力，正好是 {company} 的 {position} 岗位需要的，因为...
```

## 定制话术（核心 6-10 问）

### 必须定制的高频问题

针对该用户背景，逐一定制以下回答（每个回答 1-2 分钟，可直接背诵）：

1. **自我介绍**（30 秒版 + 1 分钟版）
2. **为什么选择我们公司**（结合公司战略 + 个人匹配）
3. **为什么从 X 转到 Y**（如有转型需求）
4. **你最大的优势是什么**（呼应三大优势）
5. **你最大的不足是什么**（必须真诚但有补救）
6. **期望薪资**（基于薪酬调研给区间）

### 该岗位的专项问题

根据岗位类型补充：

| 岗位类型 | 必备问题 |
|---------|---------|
| 销售岗 | 「你有没有客户资源能带过来」「客户说不需要怎么办」「最高签过多大单」 |
| 技术岗 | 「为什么用这个技术栈」「设计一个 XX 系统」「上一份工作技术债怎么处理」 |
| 产品岗 | 「最近用过最好的产品」「设计一款 XX 产品」「如何衡量产品成功」 |
| 运营岗 | 「如何 0 到 1 做 XX」「核心数据指标」「最近一次成功的活动」 |
| 管理岗 | 「如何带新人」「下属业绩不达标怎么办」「怎么向上汇报」 |

### 定制话术输出格式

```markdown
### Q：{完整问题}

**这道题在测什么**：
{用 1 句话点出面试官的真实意图}

**回答框架**（结构）：
1. 第一段：{建议讲什么，30 秒}
2. 第二段：{建议讲什么，30 秒}
3. 第三段：{建议讲什么，30 秒}

**完整回答**（建议背诵）：

> "{完整回答原文，注意口语化、有节奏、有数据}"

**避免的雷区**：
- 不要 {具体雷区 1}
- 不要 {具体雷区 2}
```

## 电梯演讲

定制两个版本，必须可直接背诵：

### 30 秒版（80-100 字）

适用场景：HR 初筛、CEO 终面开场、行业活动自我介绍

格式要求：
- 第 1 句：你是谁（姓名 + 一句话定位）
- 第 2 句：你做过什么（最有代表性的 1 个成就）
- 第 3 句：你为什么来（链接到这家公司）

### 1 分钟版（200-250 字）

适用场景：业务面试、HR 二面、深度沟通

格式要求：
- 开场（10 秒）：你是谁
- 经历（25 秒）：2-3 个里程碑
- 优势（15 秒）：三大核心优势
- 收尾（10 秒）：为什么是这家公司

## 反向提问精选 10 问

每场面试根据面试官身份选 3-5 个，不必全问。

**① 这个岗位是新设还是替补？前任是因为晋升还是离职？**
> 最直接的信号题。前任晋升说明这是成长岗；离职要追问原因，可能揭示团队或业务问题。

**② 这个岗位 6 个月内的核心考核指标是什么？**
> 考察公司对岗位的真实期望，也帮你判断能不能在试用期达标。

**③ 如果我入职 6 个月，做出什么样的结果，你会觉得是 A+ 表现？**
> 把"期望"具体化。展示你有结果导向思维，记住答案入职后对标执行。

**④ 过往在这个岗位做得最好的人，有什么共同特质？**
> 侧面了解胜任模型，帮你在剩余面试环节校准自我描述。

**⑤ 这个岗位最容易遇到什么卡点？优秀的人是怎么突破的？**
> 问出真实工作难度。答案本身就是你入职后的行动参考。

**⑥ 团队的跨部门协作机制是怎样的？[关键资源] 申请起来容不容易？**
> 了解实际的工作支撑，避免入职后发现"什么都要自己扛"。

**⑦ 公司对这个岗位未来 1-2 年的投入规划是怎样的？**
> 判断是战略重点还是边缘业务，直接影响你的发展空间。

**⑧ 团队是更注重过程管理还是结果导向？**
> 判断管理风格是否和你匹配，比问"加班多不多"更有信息量。

**⑨ 在 {company} 工作，最让你有成就感的事情是什么？**
> 唯一问面试官个人感受的问题。让对话变得真诚，侧面观察公司文化。

**⑩ 如果让你给即将加入的我一个建议，会是什么？**
> 压轴必问。愿意认真回答说明你给对方留下了好印象；答案本身也是最值钱的经验。

### 按面试官身份的选题建议

| 面试官身份 | 推荐优先问 | 原因 |
|----------|-----------|------|
| HR 初筛 | ②⑦⑧ | 聊岗位期望和公司方向，建立好印象 |
| 直属 Leader | ①②③⑤⑥ | 了解真实工作内容和考核，最重要 |
| 业务负责人 | ③④⑦ | 战略对齐，展示你在意业务价值 |
| HRBP / 终面 | ⑧⑨⑩ | 建立信任感，留下真诚的最后印象 |

## 面试前 3 天准备清单

```markdown

### 下午
- [ ] 把 30 秒和 1 分钟自我介绍背熟
- [ ] 录制视频自查（语速、表情、停顿）

### 晚上
- [ ] 看 1 篇创始人 / CEO 的近期访谈（YouTube 或公众号）
- [ ] 在脑子里走一遍：我面试的人最关心我什么

## T-2 天

### 上午
- [ ] 把 6 个 STAR 案例分别讲一遍（对着镜子讲）
- [ ] 找朋友模拟面试（至少 30 分钟）

### 下午
- [ ] 把所有定制话术口头讲一遍，记不住的标记出来
- [ ] 准备纸质 / 电子简历各 3 份

### 晚上
- [ ] 从 10 个反向提问中，按明天的面试官身份选出 3-5 个
- [ ] 早睡

## T-1 天（面试前一天）

### 上午
- [ ] 看 1 篇该公司的最新新闻（确保面试时知道最新动态）
- [ ] 检查交通路线 / 视频面试设备

### 下午
- [ ] 整体过 1 遍调研报告，重点关注公司差异化优势
- [ ] 准备好穿的衣服

### 晚上
- [ ] 不要再补习，让大脑放松
- [ ] 11 点前睡

## 面试当天

### 出发前
- [ ] 提前 30 分钟到（线下）/ 提前 10 分钟开机（线上）
- [ ] 带：简历、纸笔、水

### 候场
- [ ] 在心里把 30 秒自我介绍默背 1 遍
- [ ] 深呼吸 3 次

### 进入面试
- [ ] 微笑、握手、自报姓名
- [ ] 全程保持 70% 的眼神接触
- [ ] 每个回答控制在 90 秒内
- [ ] 必问反向提问

### 面试后
- [ ] 24 小时内发感谢信（邮件或短信）
- [ ] 复盘：讲得好的、卡住的、对方反应
```


### Skill 执行流程（统一）

无论哪种调用形式，Skill 都按以下顺序执行：

1. **解析输入**（第〇步）
   - URL → WebFetch
   - PDF → Read
   - 文字 → 直接解析
   - 失败的字段 → 明确反馈给用户

2. **结构化提取**
   - `parsed_jd`：核心职责、硬技能、软技能、加分项
   - `parsed_user`：基本信息、职业路径、项目、技能、动机
   - `parsed_company`：从 URL 抓取的额外公司信息

3. **简化版调研基础**（公司、产品、岗位、薪酬）

4. **匹配度分析**（JD vs 用户背景，逐项打分）

5. **三大优势提炼**（基于 parsed_user 推断）

6. **6 个 STAR 案例**（基于 parsed_user.projects）

7. **8-10 个核心问题定制话术**

8. **30 秒 + 1 分钟电梯演讲**

9. **16 个反向提问 + 3 天准备清单**

10. **生成 Markdown，可选 PDF 转换**
