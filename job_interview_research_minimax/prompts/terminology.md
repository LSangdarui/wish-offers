## 一、专业术语详解

### AI 大模型相关

| 术语 | 全称 | 通俗解释 |
|------|------|----------|
| **AGI** | Artificial General Intelligence | 通用人工智能，像人类一样什么都能干的 AI |
| **MaaS** | Model as a Service | 模型即服务，像租云服务器一样"租"AI 能力 |
| **LLM** | Large Language Model | 大语言模型，如 ChatGPT、Minimax-M2 |
| **MoE** | Mixture of Experts | 混合专家模型，把不同专家组合，需要时只叫醒部分 |
| **Token** | 模型处理的最小单位 | 约等于字数，1 汉字≈1-2 token |
| **多模态** | Multi-modal | 能处理文本+语音+视频+音乐等多种内容 |
| **长上下文** | Long Context | 模型能记住的上下文长度，Minimax 支持 100 万 token |
| **推理** | Inference | 模型根据输入生成输出的过程 |
| **训练** | Training | 用数据教模型学习的过程 |
| **POC** | Proof of Concept | 概念验证，给客户做试用 Demo |
| **API** | Application Programming Interface | 软件间的"插座"，让系统互相调用 |
| **SLA** | Service Level Agreement | 服务等级协议，承诺系统可用性 |
| **ROI** | Return on Investment | 投资回报率，花 1 块赚回几块 |
| **Agent** | 智能体 | 能自主规划、使用工具的 AI 系统 |

### 云计算相关

| 术语 | 全称 | 通俗解释 |
|------|------|----------|
| **IaaS** | Infrastructure as a Service | 基础设施即服务，租服务器 |
| **PaaS** | Platform as a Service | 平台即服务，租开发平台 |
| **SaaS** | Software as a Service | 软件即服务，直接租软件用 |
| **K8s** | Kubernetes | 容器编排系统，管理成千上万个容器 |
| **Docker** | 容器化平台 | 把应用和依赖打包成"集装箱" |
| **FinOps** | Financial Operations | 财务运维，云成本优化 |
| **TCO** | Total Cost of Ownership | 总拥有成本，买+用+维护的全部成本 |

### 销售/商业相关

| 术语 | 全称 | 通俗解释 |
|------|------|----------|
| **KA** | Key Account | 重点客户/大客户 |
| **ToB** | To Business | 面向企业客户 |
| **ToC** | To Consumer | 面向个人消费者 |
| **SQL** | Sales Qualified Lead | 销售合格线索 |

## 三、产品矩阵详解

### 核心产品

| 产品 | 定位 | 亮点 | 应用场景 |
|------|------|------|----------|
| **MiniMax-M1** | 开源推理模型 | 全球首个开源大规模混合架构推理模型，100 万 token 长上下文，SWE-bench 56% | 代码生成、长文档分析 |
| **MiniMax-M2/M2.5/M2.7** | 通用大模型 | 主打"极致质价比"，核心性能对标国际一流，推理成本大幅降低 | 企业级 API 调用 |
| **Hailuo 02（海螺）** | 视频生成模型 | 可生成复杂动作视频（如"猫跳水"），挑战谷歌 Veo3 | 广告视频、内容创作 |
| **Speech 02** | 语音模型 | 国际榜单登顶全球第一 | 语音合成、智能客服 |
| **Music 2.0** | 音乐模型 | 精准捕捉人声情绪 | 音乐创作、游戏配乐 |
| **MiniMax Agent** | 通用智能体 | 具备复杂 Agent 能力，支持多轮高保真编辑 | 自动化任务、工作流 |
| **Talkie/星野** | AI 社交应用 | C 端出海产品，海外 2.12 亿用户 | AI 陪伴、角色扮演 |

### 核心技术优势

**1. MoE 混合架构**
- 每次只激活部分参数（如 10%），推理成本低
- M1 强化学习训练仅用 53.5 万美元（同行需千万美元）

**2. 100 万 Token 长上下文**
- 可处理整本《三体》（约 50 万字）
- 解锁法律合同审查、代码库分析、复杂 Agent 工作流等场景

**3. 全模态能力**
- 国内唯一同时做到文本、语音、视频、音乐四大赛道第一梯队的公司

## 五、MaaS 行业深度分析

### 什么是 MaaS？

**定义**：Model as a Service，模型即服务

**类比理解**：
- 以前企业想用 AI = 自己买 GPU + 招算法工程师 + 训练模型（自己盖房子）
- MaaS = 调用 API 按量付费（租公寓，拎包入住）

### 收费模式

| 模式 | 说明 | 适用客户 |
|------|------|----------|
| 按 Token 计费 | 用多少付多少 | 用量波动大的中小客户 |
| 订阅制 | 包月/包年 | 用量稳定的成熟客户 |
| 私有化部署 | 部署在客户自有服务器 | 数据敏感的大客户 |
| 按效果付费 | 按业务效果付费 | 效果可量化的场景 |

### 市场规模

- **2024 年中国 MaaS 市场**：7.1 亿元
- **同比增长**：215.7%（爆发式增长）
- **主要行业**：金融 49%、政务 8%、娱乐/媒体各 7%

### 企业客户 AI 落地痛点

| 层面 | 痛点 | Minimax 解决方案 |
|------|------|------------------|
| 技术 | 模型幻觉、集成难度、效果不稳定 | 私有化部署、长上下文能力 |
| 成本 | 算力成本高、ROI 难量化 | 极致性价比、TCO 分析 |
| 安全 | 数据隐私、合规要求 | 私有化部署、安全审计 |
| 组织 | 员工抵触、人才缺乏 | 培训支持、客户成功服务 |
| 认知 | 期望过高、场景选择困难 | 行业解决方案、ROI 论证 |
